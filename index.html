06-组件通信

Posted on 2019-04-14 | Edited on 2019-04-15

网络通信

组件挂载阶段通信

组件挂载时通信一般在componentDidMount中进行服务器的通信请求

componentDidMount中执行服务器通信可以保证获取到数据时，组件已经处于挂载状态，这时即使直接操作DOM也是安全的，而componentWillMount无法保证这一点.

当组件在服务器端渲染时，componentWillMount会被调用两次，一次是在服务器端，另一次是在浏览器端，而componentDidMount能保证在任何情况下只会被调用一次，从而不会发送多余的数据请求。

虽然在构造函数中执行通信请求一般也会正常工作，但是构造函数的意义是执行组件的初始化工作，并不适合做数据请求这类有’副作用’的工作因此不推荐在构造函数中进行数据请求。

组件更新阶段通信

组件在更新阶段常常需要再次与服务器通信，获取服务器上的最近数据。例如，组件需要以props中的某个属性作为与服务器通信的请求参数，当这个属性值发生更新时，组件自然需要重新与服务器通信。因此，componentWillReceiveProps最适合

组件通信

父子组件通信

父子组件通信是最常见的通信方式，父组件向子组件传递信息的方式是通过父组件向子组件的props传递数据完成的。而子组件向父组件传递信息的方式依然是props，父组件可以通过子组件的props传递给子组件一个回调函数，子组件在需要改变父组件数据时，调用这个回调函数即可。

    例如：

决定是否作为state的属性的最简单判断方法，可以通过查看组件的render方法是否使用了该属性

决定属性是否为state的依据

1.变量是否通过props从父组件获取？如果是，则不是state属性

2.这个变量是否在组件的整个生命周期中保持不变？如果是，则不是state属性

3.这个变量是否可以通过其他状态(state)或者属性(props)计算获取？如果是，则不是state属性

4.这个变量是否在组件的render方法中使用？如果不是，则不是state属性

以上情况应考虑作为普通属性

正确修改state

state可以通过this.state.[属性名]获取，但修改时需要注意

1.不能直接修改state
2.state的更新时异步的
3.state的更新是一个合并的过程

state与不可变对象

React官方建议把 state 当做不可变对象，一方面，直接修改this.state组件不会重新render，另一方面，state中包含的所有状态都应该是不可变对象。当state中的某个状态发生变化时，应该重新创建这个状态对象而不是直接修改状态中的属性。当状态发生改变时，根据状态的类型可以分成以下三种情况：

1.状态的类型是不可变类型(数字、字符串、布尔值、null、undefined)

直接赋新值即可

2.状态的类型是数组

方法1：使用preState、concat创建新数组
方法2： ES6 spread syntax

当从数组中截取部分元素作为新状态时，可使用数组的slice方法

当从books中保留过滤后的元素，作为新状态，可使用数组的filter方法

不要使用push、pop、shift、unshift、splice等方法修改数组类型的状态，因为这些方法都是在原数组的基础上修改的，而concat、slice、filter会返回一个新的数组

3.状态类型是普通对象

1.使用ES6的Object.assgin方法

2.使用对象扩展语法(object spread properties)

与数组类似，应避免使用会直接修改原对象的方法，而是直接返回一个新对象的方法。
