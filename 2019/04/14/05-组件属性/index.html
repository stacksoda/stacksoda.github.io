组件state

state必须能代表一个组件UI呈现的完整状态集，且应该为最小状态集

state的所代表的完整状态集中的数据又可以分为两类数据

  1. 用作渲染时使用到的数据的来源
  2. 用作组件UI是否渲染的判断依据
    例如：

决定是否作为state的属性的最简单判断方法，可以通过查看组件的render方法是否使用了该属性

决定属性是否为state的依据

1.变量是否通过props从父组件获取？如果是，则不是state属性

2.这个变量是否在组件的整个生命周期中保持不变？如果是，则不是state属性

3.这个变量是否可以通过其他状态(state)或者属性(props)计算获取？如果是，则不是state属性

4.这个变量是否在组件的render方法中使用？如果不是，则不是state属性

以上情况应考虑作为普通属性

正确修改state

state可以通过this.state.[属性名]获取，但修改时需要注意

1.不能直接修改state
2.state的更新时异步的
3.state的更新是一个合并的过程

state与不可变对象

React官方建议把 state 当做不可变对象，一方面，直接修改this.state组件不会重新render，另一方面，state中包含的所有状态都应该是不可变对象。当state中的某个状态发生变化时，应该重新创建这个状态对象而不是直接修改状态中的属性。当状态发生改变时，根据状态的类型可以分成以下三种情况：

1.状态的类型是不可变类型(数字、字符串、布尔值、null、undefined)

直接赋新值即可

2.状态的类型是数组

方法1：使用preState、concat创建新数组
方法2： ES6 spread syntax

当从数组中截取部分元素作为新状态时，可使用数组的slice方法

当从books中保留过滤后的元素，作为新状态，可使用数组的filter方法

不要使用push、pop、shift、unshift、splice等方法修改数组类型的状态，因为这些方法都是在原数组的基础上修改的，而concat、slice、filter会返回一个新的数组

3.状态类型是普通对象

1.使用ES6的Object.assgin方法

2.使用对象扩展语法(object spread properties)

与数组类似，应避免使用会直接修改原对象的方法，而是直接返回一个新对象的方法。
