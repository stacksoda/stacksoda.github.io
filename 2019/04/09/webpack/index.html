管理资源

加载 css

webpack 使用 style-loader和css-loader来使 JavaScript 能够 import 一个 CSS 文件

webpack.config.js中

其中，css-loader 用于使类似@import 和 url(…)的方法实现 require 的功能，style-loader 用于将所有计算后的样式加入页面中

实际运行中当遇到后缀为.css 的文件,webpack先使用css-loader解析这个文件，遇到@import 语句会将对应文件引入并解析，最后计算完的 css 会使用style-loader生成一个内容为最终完成解析的 style 标签,放到 head 标签里
> webpack中 loader 是有解析顺序的，从右向左的执行，因此 css-loader 应在 style-loader 的后面

加载图片

webpack 借助 file-loader 来实现对图片资源的加载,每一个 loader 都如加载 css 资源时那样在module.rules下声明 形式类似于：

此时通过import(在.js文件中)方式引入的文件便可以通过file-loader进行加载,而在css中的url(./image.png)则是通过css-loader进行的类似于file-loader的加载处理。

官方解释如下:

当你 import MyImage from './my-image.png'，该图像将被处理并添加到 output 目录，_并且_ MyImage 变量将包含该图像在处理后的最终 url。当使用 css-loader 时，CSS 中的 url('./my-image.png') 会使用类似的过程去处理。loader 会识别这是一个本地文件，并将 './my-image.png' 路径，替换为输出目录中图像的最终路径。html-loader 以相同的方式处理 <img src="./my-image.png" />。
